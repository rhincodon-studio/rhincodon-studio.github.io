RKE2 高可用叢集（HA）

1. 架構說明​

RKE2 官方建議的高可用架構至少需要 3 台 Control Plane（含 etcd） 搭配 VIP / LoadBalancer。

範例拓撲​

節點角色IP
cp-1control-plane + etcd192.168.1.10
cp-2control-plane + etcd192.168.1.20
cp-3control-plane + etcd192.168.1.30
worker-1worker192.168.1.40
VIP虛擬 IP192.168.1.100

Load Balancer 需要轉發：​

  • 6443 → Kubernetes API
  • 9345 → RKE2 Supervisor

2. 安裝 Control Plane（每台 Server 都要執行）​

2.1 安裝 RKE2​

curl -sL https://get.rke2.io | INSTALL_RKE2_VERSION=v1.27.12+rke2r1 sh

2.2 建立 Control Plane 設定檔​

路徑：/etc/rancher/rke2/config.yaml

server: https://<VIP>:9345
token: "<your-cluster-token>"
tls-san:
  - <VIP>
  - <LB-domain>

2.3 啟動 RKE2 Server​

systemctl enable rke2-server
systemctl start rke2-server

3. 安裝 Worker Nodes​

3.1 安裝 Agent​

curl -sL https://get.rke2.io | INSTALL_RKE2_TYPE="agent" sh

3.2 建立 Worker 設定檔​

/etc/rancher/rke2/config.yaml：

server: https://<VIP>:9345
token: "<your-token>"

3.3 啟動 agent​

systemctl enable rke2-agent
systemctl start rke2-agent

4. Load Balancer（HAProxy / Keepalived）​

4.1 Keepalived（提供 VIP）​

vrrp_instance VI_1 {
  state MASTER
  interface eth0
  virtual_router_id 55
  priority 200
  advert_int 1
  virtual_ipaddress {
    192.168.1.100
  }
}

4.2 HAProxy（轉發 6443 / 9345）​

frontend k8s_api
  bind *:6443
  default_backend k8s_api_backend

backend k8s_api_backend
  balance roundrobin
  server cp-1 192.168.1.10:6443 check
  server cp-2 192.168.1.20:6443 check
  server cp-3 192.168.1.30:6443 check

frontend supervisor
  bind *:9345
  default_backend supervisor_backend

backend supervisor_backend
  balance roundrobin
  server cp-1 192.168.1.10:9345 check
  server cp-2 192.168.1.20:9345 check
  server cp-3 192.168.1.30:9345 check

5. 驗證叢集狀態​

5.1 檢查節點​

kubectl get nodes -o wide

5.2 檢查所有 Pod​

kubectl get pods -A

5.3 檢查 etcd 成員​

/var/lib/rancher/rke2/bin/etcdctl   --cacert=/var/lib/rancher/rke2/server/tls/etcd/server-ca.crt   --cert=/var/lib/rancher/rke2/server/tls/etcd/server.crt   --key=/var/lib/rancher/rke2/server/tls/etcd/server.key   member list

6. Troubleshooting（常見問題排查）​

問題 1：connection refused 127.0.0.1:2379​

原因：etcd 尚未啟動
查看：

journalctl -u rke2-server -f

問題 2：x509: certificate is valid for ... not VIP​

加入：

tls-san:
  - <VIP>

問題 3：Worker 無法加入​

LB 必須同時處理 6443 與 9345。

問題 4：etcd quorum lost​

三台 control-plane 至少要活著兩台。

檢查：

etcdctl endpoint health
